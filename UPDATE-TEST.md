# Testing Auto-Update (v0.1.0 → v0.2.0)

## Prerequisites
- v0.1.0 already installed on your machine
- Administrator PowerShell
- GitHub CLI (`gh`) authenticated

## Step 1: Clean winCodeSign cache
```powershell
Remove-Item -Recurse -Force "$env:LOCALAPPDATA\electron-builder\Cache\winCodeSign" -ErrorAction SilentlyContinue
```

## Step 2: Pull latest and build v0.2.0
```powershell
cd C:\Users\Ryzz3nn\Documents\aDevProjects\.Apps\rLauncher
git pull
npm run dist:win
```

## Step 3: Create GitHub Release
```powershell
gh release create v0.2.0 "release\rLauncher Setup 0.2.0.exe" "release\latest.yml" "release\rLauncher-0.2.0-portable.exe" --title "v0.2.0" --notes "Test update"
```

## Step 4: Test
1. Open the installed rLauncher (v0.1.0)
2. Go to **Settings**
3. Should show "Update v0.2.0 available" (checks automatically after 5 seconds)
4. Click **Download** → progress bar fills
5. Click **Restart & Update** → app closes, installs update, reopens as v0.2.0
6. Verify version shows `v0.2.0` in Settings

## Notes
- `latest.yml` is auto-generated by electron-builder in the `release/` folder
- Both the Setup .exe and `latest.yml` must be in the GitHub Release for auto-update to work
- All user data (`%APPDATA%/rlauncher/`) persists across updates
